Jira Basics

Q1-15
1 What is Jira?
Jira is a project management and issue tracking tool developed by Atlassian. It's widely used for Agile development (Scrum/Kanban), bug tracking, and task management.
2 What are the main products in the Jira family?
Jira Software: For software development teams
Jira Service Management: For IT service management
Jira Work Management: For business teams
Jira Align: For enterprise agile planning
3 What is an Issue in Jira?
An issue is the fundamental work item in Jira. It can be a bug, task, story, epic, sub-task, or custom issue type.
4 What are the default issue types in Jira?
Epic: Large body of work
Story: User requirement
Task: Work item
Bug: Defect
Sub-task: Breakdown of a parent issue
5 What is the difference between a Task and a Story?
Story: Describes a feature from user's perspective ("As a user...")
Task: Generic work item not necessarily user-facing
6 What is an Epic in Jira?
A large user story that can be broken down into smaller stories. Epics span multiple sprints and represent significant features or initiatives.
7 What are Components in Jira?
Components are sub-sections of a project used to group issues by functionality, module, or team (e.g., UI, Backend, Database).
8 What are Labels in Jira?
Labels are keywords or tags that can be added to issues for flexible categorization and filtering. Unlike components, labels can be created on-the-fly.
9 What is Priority in Jira?
Priority indicates the importance/urgency of an issue. Default priorities: Highest, High, Medium, Low, Lowest.
10 What is Status in Jira?
Status represents where an issue is in the workflow (e.g., To Do, In Progress, Done). Statuses are defined in workflows.
11 What is Resolution in Jira?
Resolution indicates how an issue was closed: Fixed, Won't Fix, Duplicate, Cannot Reproduce, Done, etc.
12 What are Story Points?
A relative measure of effort/complexity for user stories. Used in Agile estimation (often using Fibonacci: 1, 2, 3, 5, 8, 13...).
13 What is a Filter in Jira?
A saved search query (usually JQL) that can be reused and shared with team members.
14 What is a Dashboard in Jira?
A customizable page displaying gadgets (charts, filters, statistics) providing project insights at a glance.
15 What are Jira Permissions?
Permissions control what users can do in a project (create issues, edit issues, delete, admin, etc.). Managed through permission schemes.

Projects & Configuration

Q16-30
16 What is a Jira Project?
A project is a collection of issues. It has a unique key (e.g., PROJ), workflows, permissions, and can be either team-managed or company-managed.
17 What's the difference between Company-managed and Team-managed projects?
Company-managed: Shared configuration, more complex, requires admin rights
Team-managed: Simplified, independent configuration, team can self-manage
18 What is a Project Key?
A unique identifier (2-10 uppercase letters) used as a prefix for issue keys. Example: PROJ-123.
19 What are Schemes in Jira?
Reusable configuration sets: - Workflow Scheme: Maps issue types to workflows
- Permission Scheme: Defines user permissions
- Notification Scheme: Defines email notifications
- Issue Type Scheme: Defines available issue types
20 What is a Custom Field?
Additional fields created beyond default Jira fields (e.g., Test Environment, Customer Name). Can be text, dropdown, date, etc.
21 What are Project Roles?
Flexible way to group users within a project (Administrators, Developers, Users). Used in permission schemes.
22 What is an Issue Link?
A relationship between two issues (Blocks, Duplicates, Relates to, Caused by). Helps track dependencies.
23 What is a Version in Jira?
Represents a release of your product. Issues can be assigned to versions for release planning and tracking.
24 What is the difference between Fix Version and Affects Version?
Affects Version: Version where bug was found
Fix Version: Version where fix will be released
25 What are Watchers in Jira?
Users who want to receive notifications about issue updates without being assigned to it.
26 What is Time Tracking in Jira?
Feature to log work hours, track original estimates, remaining estimates, and time spent on issues.
27 What are Sub-tasks?
Smaller pieces of work that are part of a parent issue. Help break down complex tasks.
28 What is Bulk Change in Jira?
Feature to edit, transition, or delete multiple issues at once using search results.
29 What are Issue Collectors?
Widget embedded in websites allowing users to create Jira issues directly from web pages.
30 What is Jira Administration?
System-level configuration including user management, global permissions, custom fields, schemes, and system settings. Requires Jira Administrator permission.

Workflows

Q31-45
31 What is a Workflow in Jira?
A workflow defines the lifecycle of an issue through various statuses (To Do ‚Üí In Progress ‚Üí Done). It consists of statuses and transitions.
32 What are Workflow Statuses?
The stages an issue passes through (Open, In Progress, Resolved, Closed). Statuses belong to categories: To Do, In Progress, Done.
33 What are Workflow Transitions?
Links between statuses that define how an issue moves through the workflow (Start Progress, Resolve, Reopen).
34 What are Workflow Conditions?
Rules that control WHO can execute a transition (e.g., only assignee can resolve, only developers can start progress).
35 What are Workflow Validators?
Check if the transition can proceed (e.g., all sub-tasks must be resolved, required fields must be filled).
36 What are Workflow Post Functions?
Actions that happen AFTER a transition (assign to current user, add comment, send notification, update field).
37 What is a Global Transition?
A transition available from ANY status in the workflow (e.g., Close issue from any state).
38 Can you have different workflows for different issue types?
Yes! Workflow schemes map issue types to specific workflows. Bugs might have a different workflow than Stories.
39 What is the default Jira workflow?
Simple workflow: Open ‚Üí In Progress ‚Üí Resolved ‚Üí Closed (with Reopen option).
40 What is workflow Draft mode?
Allows editing active workflows without immediately affecting projects. Publish when ready.
41 What happens to issues when you change a workflow?
Issues in statuses that don't exist in new workflow need to be migrated. Jira provides a migration wizard.
42 What is Transition Screen?
A form shown during a transition where users can edit fields (e.g., add resolution when resolving).
43 What are Workflow Properties?
Key-value pairs that can be attached to workflows, statuses, or transitions for advanced configurations.
44 Can workflows be shared across projects?
Yes! Workflows can be reused in multiple projects through workflow schemes.
45 What is a Looping Transition?
A transition that returns an issue to the same status (e.g., Needs More Info ‚Üí Open ‚Üí Needs More Info).

Agile & Boards

Q46-60
46 What is a Scrum Board in Jira?
An Agile board for teams using Scrum methodology with sprints, backlog, and burndown charts.
47 What is a Kanban Board in Jira?
An Agile board for continuous flow work without sprints. Focuses on WIP limits and cycle time.
48 What's the difference between Scrum and Kanban boards?
Scrum: Time-boxed sprints, sprint planning, velocity
Kanban: Continuous flow, WIP limits, cycle time
49 What is a Sprint in Jira?
A time-boxed iteration (usually 1-4 weeks) where a team commits to completing a set of issues.
50 What is the Backlog in Jira?
A prioritized list of issues waiting to be worked on. Product Owners manage the backlog priority.
51 What is Sprint Planning?
Meeting where team selects issues from backlog to work on in the upcoming sprint.
52 What is a Burndown Chart?
Graph showing remaining work (story points/hours) over sprint duration. Helps track if team is on track.
53 What is Velocity in Jira?
Average story points completed per sprint. Used for capacity planning in future sprints.
54 What is WIP (Work In Progress) Limit?
Maximum number of issues allowed in a Kanban column. Prevents overloading and improves flow.
55 What is a Release in Jira?
Collection of versions shipped together. Releases can span multiple sprints and projects.
56 What is Cumulative Flow Diagram?
Chart showing work distribution across workflow states over time. Available in Kanban boards.
57 What is Cycle Time?
Time an issue spends from "In Progress" to "Done". Key metric for Kanban teams.
58 What are Quick Filters in Jira boards?
One-click filters on boards to show specific issues (My Issues, Only Bugs, Current Sprint).
59 What is Epic in Agile board?
Large work items shown as swimlanes on boards. Track progress of multiple stories under one epic.
60 What is Sprint Report?
Report showing sprint completion status, velocity, issues completed/not completed, and scope changes.

JQL (Jira Query Language)

Q61-75
61 What is JQL?
Jira Query Language - a structured query language for searching issues. Similar to SQL but for Jira.
62 What is the basic syntax of JQL?
field operator value
Example: project = PROJ AND status = "In Progress"
63 What are common JQL operators?
= equals
!= not equals
IN matches any value
NOT IN doesn't match
~ contains text
IS EMPTY / IS NOT EMPTY
64 How do you find all issues assigned to you?
assignee = currentUser()
65 How do you find all open bugs?
type = Bug AND status != Done
66 How do you find issues created in the last week?
created >= -7d
67 How do you find issues updated today?
updated >= startOfDay()
68 How do you find unassigned issues?
assignee IS EMPTY
69 How do you combine multiple conditions in JQL?
AND - both conditions must be true
OR - either condition true
Example: priority = High AND (status = Open OR status = "In Progress")
70 How do you sort results in JQL?
ORDER BY field ASC/DESC
Example: status = Open ORDER BY priority DESC, created ASC
71 What are JQL Functions?
Built-in functions for dynamic queries:
currentUser() - logged-in user
now() - current date/time
startOfDay() - beginning of today
endOfWeek() - end of current week
72 How do you search by text in description or comments?
text ~ "search term" (searches summary, description, comments)
73 How do you find issues in current sprint?
sprint in openSprints()
74 How do you find overdue issues?
duedate < now() AND status != Done
75 Can you use JQL in automation rules?
Yes! JQL is used in automation triggers and conditions to find matching issues.

Advanced Topics

Q76-85
76 What is Jira Automation?
No-code rules that automate tasks (auto-assign, send notifications, transition issues) based on triggers and conditions.
77 What are the components of an Automation Rule?
Trigger: When rule runs (issue created, field changed)
Condition: If criteria met (if priority = High)
Action: What to do (assign, comment, transition)
78 What is Jira REST API?
HTTP API for programmatic access to Jira (create issues, update fields, search). Used for integrations.
79 What are Jira Apps/Plugins?
Extensions from Atlassian Marketplace that add functionality (Tempo for time tracking, Zephyr for test management).
80 What is Confluence and how does it integrate with Jira?
Confluence is Atlassian's wiki tool. You can link Jira issues to Confluence pages and display Jira macros in Confluence.
81 What is Bitbucket and how does it integrate with Jira?
Bitbucket is Atlassian's Git repository tool. Commits/PRs can reference Jira issues, showing code changes in issues.
82 What is Jira Service Management (formerly Service Desk)?
ITSM tool for managing service requests, incidents, and customer support tickets with SLA tracking.
83 What are SLAs in Jira?
Service Level Agreements - commitments for response/resolution times. Tracked with SLA metrics and escalations.
84 What is Portfolio for Jira (now Advanced Roadmaps)?
Tool for portfolio planning, roadmapping, capacity planning, and dependency management across multiple teams.
85 What are best practices for Jira performance?
- Archive old projects
- Limit custom fields
- Optimize JQL queries
- Use dashboards wisely
- Regular cleanup of old issues
- Limit automation rules
