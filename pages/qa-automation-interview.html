Automation Fundamentals

Q1-15
1 What is Test Automation?
The use of software tools and scripts to execute test cases automatically, reducing manual effort and increasing testing efficiency.
2 What are the benefits of automation testing?
- Faster execution
- Reusability of tests
- Better test coverage
- Early bug detection
- Cost-effective for regression
- Parallel execution
- Consistent results
3 When should you automate tests?
- Repetitive test cases
- Regression test suites
- Smoke/sanity tests
- Data-driven scenarios
- Performance testing
- Cross-browser testing
- Stable features
4 When NOT to automate?
- One-time tests
- Unstable features
- Complex UI tests
- Usability testing
- Ad-hoc/exploratory testing
- When ROI is low
5 What is an Automation Framework?
A structured set of guidelines, coding standards, concepts, processes, and tools that support test automation. Provides reusability, maintainability, and scalability.
6 What are types of automation frameworks?
1. Linear/Record & Playback: Basic, no reusability
2. Modular: Divided into modules
3. Data-Driven: Test data separated
4. Keyword-Driven: Actions as keywords
5. Hybrid: Combination of above
6. BDD: Behavior-driven (Cucumber)
7 What is Data-Driven Framework?
Framework where test data is separated from test scripts and stored externally (Excel, CSV, JSON, DB). Same test runs with different data sets.
8 What is Keyword-Driven Framework?
Framework where test cases are written using keywords that represent actions. Non-technical users can create tests.
9 What is Page Object Model (POM)?
Design pattern where each page is a class with page elements and methods. Separates page logic from test logic, improving maintainability.
10 What is Test Pyramid?
Testing strategy visualization:
- Bottom (Most): Unit Tests
- Middle: Integration/API Tests
- Top (Least): UI Tests
More lower-level tests, fewer UI tests (faster, cheaper)
11 What is ROI in test automation?
Return on Investment - measuring benefits vs costs of automation. Consider: development time, maintenance, execution time savings, bug detection.
12 What is flaky test?
Test that sometimes passes and sometimes fails without code changes. Causes: timing issues, environment dependencies, race conditions.
13 How do you handle flaky tests?
- Add proper waits
- Make tests independent
- Handle asynchronous operations
- Use stable locators
- Implement retry mechanism
- Fix root cause, not symptoms
14 What is test maintainability?
Ease of updating tests when application changes. Good practices: POM, reusable functions, clear naming, minimal duplication.
15 What is test coverage in automation?
Percentage of application features/code covered by automated tests. Aim for critical paths, not 100% automation.

Automation Tools & Technologies

Q16-30
16 Compare Selenium, Playwright, and Cypress
Selenium: Mature, multi-language, all browsers, slower
Playwright: Modern, fast, auto-waits, multi-browser
Cypress: JavaScript only, fast, Chrome-focused, great DX
17 What is Appium?
Open-source tool for mobile app automation (iOS, Android). Uses WebDriver protocol, supports native, hybrid, and mobile web apps.
18 What is RestAssured?
Java library for REST API automation. Provides DSL for HTTP requests, response validation, authentication, and JSON/XML parsing.
19 What is Postman?
API testing tool with GUI for sending requests, collections, environments, automated testing with Newman.
20 What is JMeter?
Performance testing tool for load testing web applications, APIs, databases. Simulates multiple users and measures response times.
21 What is TestNG?
Testing framework for Java with annotations, parallel execution, test grouping, dependencies, data providers, and reporting.
22 What is JUnit?
Unit testing framework for Java. Provides annotations (@Test, @Before, @After), assertions, and test runners.
23 What is Cucumber?
BDD tool using Gherkin syntax (Given-When-Then). Bridges communication between technical and non-technical team members.
24 What is Maven?
Build automation and dependency management tool. Uses pom.xml to manage project dependencies, plugins, and build lifecycle.
25 What is Gradle?
Build automation tool similar to Maven but uses Groovy/Kotlin DSL. Faster builds, better dependency management.
26 What is Docker in test automation?
Containerization platform for consistent test environments. Run tests in isolated containers, easy setup, cross-platform compatibility.
27 What is BrowserStack/Sauce Labs?
Cloud-based platforms for cross-browser testing. Access to multiple browser/OS combinations without maintaining infrastructure.
28 What is Allure Report?
Test reporting framework with rich visualizations, test history, attachments, categorization, and trend analysis.
29 What is Extent Reports?
HTML reporting library for test results with dashboard, charts, screenshots, and detailed test information.
30 What is WebDriverManager?
Library that automatically downloads and manages browser drivers. Eliminates manual driver management.

Framework Design & Architecture

Q31-45
31 What are the components of a good automation framework?
- Test scripts
- Page Objects
- Test data files
- Configuration files
- Utility/Helper classes
- Reporting mechanism
- Logging
- Exception handling
32 What is a Singleton pattern in automation?
Design pattern ensuring only one instance of a class exists (e.g., WebDriver instance). Useful for shared resources.
33 What is Factory pattern in automation?
Design pattern for creating objects without specifying exact class. Used for browser initialization (ChromeDriver, FirefoxDriver).
34 How do you handle test data?
- Excel/CSV files
- JSON/XML files
- Property files
- Database
- Test data generators
- Environment-specific configs
35 How do you handle configuration management?
Use properties/YAML files for environment URLs, credentials, timeouts. Separate configs for dev/staging/prod.
36 What is dependency injection in testing?
Providing dependencies (WebDriver, data) from outside rather than creating inside. Makes tests more flexible and testable.
37 How do you implement logging?
Use Log4j/SLF4J to log test execution steps, errors, debug info. Different log levels (INFO, DEBUG, ERROR).
38 How do you handle screenshots in framework?
- Capture on failure
- Store with test name + timestamp
- Attach to reports
- Create screenshot utility class
- Configure storage location
39 What is Base Test class?
Parent class containing setup/teardown methods, common utilities. All test classes extend it for shared functionality.
40 How do you organize test suites?
- By feature/module
- By priority (smoke, regression)
- By test type (API, UI)
- Use TestNG XML or JUnit suites
- Tag-based execution
41 How do you handle parallel execution?
- TestNG parallel execution
- ThreadLocal for WebDriver
- Ensure test independence
- Manage shared resources
- Configure thread count
42 What is ThreadLocal in Selenium?
Provides thread-local variables. Each thread has its own WebDriver instance for parallel execution without interference.
43 How do you implement retry mechanism?
Use TestNG IRetryAnalyzer or similar to retry failed tests automatically. Set max retry count to handle flaky tests.
44 What is test independence?
Each test should run independently without depending on other tests. No shared state, proper setup/cleanup.
45 How do you version control automation framework?
Use Git for source control. Follow branching strategy, code reviews, commit best practices, .gitignore for sensitive data.

CI/CD Integration

Q46-60
46 What is CI/CD?
CI: Continuous Integration - automatically build/test on code commit
CD: Continuous Deployment - automatically deploy to production
47 What is Jenkins?
Open-source CI/CD tool for automating builds, tests, and deployments. Supports plugins, pipelines, scheduled jobs.
48 How do you integrate Selenium with Jenkins?
1. Install Jenkins
2. Configure Maven/Gradle
3. Create Jenkins job
4. Add build triggers
5. Execute test suite
6. Publish test reports
49 What is Jenkins Pipeline?
Pipeline as code using Jenkinsfile. Defines stages (checkout, build, test, deploy) in declarative or scripted syntax.
50 What is GitHub Actions?
CI/CD platform integrated with GitHub. Define workflows in YAML, trigger on push/PR, matrix builds for multiple environments.
51 What is GitLab CI/CD?
Built-in CI/CD in GitLab using .gitlab-ci.yml. Pipelines, stages, jobs, runners for automated testing.
52 What are build triggers?
- SCM polling (check for changes)
- Webhook (on code push)
- Scheduled (cron jobs)
- Manual trigger
- Upstream/downstream jobs
53 How do you run tests in CI/CD?
- Configure test environment
- Use headless browsers
- Run in Docker containers
- Execute test suite via Maven/Gradle
- Publish test reports
- Notify on failures
54 How do you handle test reports in CI/CD?
- Use Jenkins plugins (HTML Publisher, Allure)
- Archive test artifacts
- Email notifications
- Slack/Teams integration
- Dashboard visualization
55 What is Docker in CI/CD?
Containers provide consistent test environments across CI/CD pipeline. Package tests with dependencies, easy scaling.
56 What is Selenium Grid in CI/CD?
Run tests in parallel across multiple machines/browsers in CI pipeline. Reduces execution time significantly.
57 How do you handle environment variables in CI/CD?
Store secrets/configs as environment variables in CI tool. Access in tests without hardcoding sensitive data.
58 What is test orchestration?
Managing test execution order, dependencies, and distribution. Tools like Jenkins, CircleCI orchestrate complex test workflows.
59 What is shift-left testing in CI/CD?
Running tests early and frequently in development cycle. Catch bugs early when they're cheaper to fix.
60 How do you handle test failures in CI/CD?
- Fail build on critical test failures
- Retry flaky tests
- Send notifications
- Capture screenshots/logs
- Mark build as unstable vs failed

Best Practices & Standards

Q61-75
61 What are automation best practices?
- Start with smoke tests
- Use POM pattern
- Implement proper waits
- Make tests independent
- Use meaningful names
- Keep tests simple
- Regular maintenance
- Version control
62 How do you name test cases?
Use descriptive names: test_loginWithValidCredentials_shouldSucceed(). Clearly state what is tested and expected outcome.
63 What is DRY principle in automation?
Don't Repeat Yourself - avoid code duplication. Create reusable methods, utilities, and page objects.
64 What is AAA pattern in testing?
Arrange: Setup test data
Act: Perform action
Assert: Verify result
65 How do you handle hard-coded values?
Avoid hardcoding! Use configuration files, constants, enums, environment variables for flexibility.
66 What is test isolation?
Tests don't share state or depend on execution order. Each test has its own setup/teardown.
67 How often should you run automated tests?
- Smoke tests: After every deploy
- Regression: Daily/nightly builds
- Full suite: Weekly
- On-demand: Before releases
68 How do you handle test data cleanup?
- Clean up in @AfterMethod/@AfterClass
- Use test databases
- Reset state between tests
- API calls for cleanup
- Avoid affecting other tests
69 What is the ideal automation test?
- Fast execution
- Reliable (not flaky)
- Independent
- Maintainable
- Clear purpose
- Good assertions
- Proper logging
70 How do you handle test documentation?
- README with setup instructions
- Framework architecture docs
- Test case descriptions
- Code comments
- Run instructions
- Troubleshooting guide
71 What is code review in automation?
Peer review of test code for quality, standards, best practices. Catch issues early, share knowledge.
72 How do you handle authentication in tests?
- Store credentials securely
- Use tokens/cookies
- API login for faster tests
- Reuse sessions
- Environment-based configs
73 What is negative testing in automation?
Testing with invalid inputs to verify error handling. Ensure app doesn't crash, shows proper error messages.
74 How do you measure automation success?
- Test coverage %
- Execution time reduction
- Bug detection rate
- Test stability (pass rate)
- ROI
- Maintenance effort
75 What is continuous testing?
Running automated tests continuously throughout the development cycle. Immediate feedback on code changes.

Advanced Topics

Q76-80
76 What is AI/ML in test automation?
- Self-healing tests (auto-fix locators)
- Visual testing (image comparison)
- Test generation from requirements
- Predictive analytics for flaky tests
- Smart test selection
77 What is contract testing?
Verifying that API provider and consumer agree on contract (structure, data types). Tools: Pact, Spring Cloud Contract.
78 What is chaos engineering in testing?
Intentionally introducing failures to test system resilience. Verify graceful degradation and recovery.
79 What is test observability?
Making test execution visible through detailed logging, metrics, traces. Understand why tests pass/fail.
80 What is the future of test automation?
- AI-powered testing
- Codeless automation
- Shift-left & shift-right
- Cloud-based testing
- Visual AI testing
- Self-healing tests
- Continuous testing in DevOps
