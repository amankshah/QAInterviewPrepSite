Git Basics

Q1-15
1 What is Git?
Git is a distributed version control system used to track changes in source code during software development. It allows multiple developers to work together on non-linear development.
2 What is the difference between Git and GitHub?
Git is a version control system (software). GitHub is a cloud-based hosting service for Git repositories (platform).
3 What is a Git repository?
A Git repository is a storage location for your project that contains all files, history, and metadata. It can be local (on your machine) or remote (on a server like GitHub).
4 How do you initialize a Git repository?
git init - Creates a new Git repository in the current directory.
5 What is the staging area in Git?
The staging area (or index) is an intermediate area where changes are collected before committing. Use git add to stage files.
6 How do you check the status of your repository?
git status - Shows the state of the working directory and staging area.
7 How do you stage files in Git?
git add <filename> - Stage a specific file
git add . - Stage all changes in the current directory
8 How do you commit changes in Git?
git commit -m "message" - Records staged changes with a descriptive message.
9 What is a commit in Git?
A commit is a snapshot of your repository at a specific point in time. It includes a unique SHA hash, author info, timestamp, and commit message.
10 How do you view commit history?
git log - Shows commit history
git log --oneline - Compact view
git log --graph - Visual branch graph
11 How do you unstage a file?
git reset <filename> or git restore --staged <filename>
12 How do you discard changes in working directory?
git checkout -- <filename> or git restore <filename>
13 What is .gitignore?
A .gitignore file specifies intentionally untracked files that Git should ignore (e.g., build files, dependencies, secrets).
14 How do you clone a repository?
git clone <repository-url> - Creates a local copy of a remote repository.
15 What is HEAD in Git?
HEAD is a pointer to the current branch reference, which points to the last commit in the current branch.

Branching & Branch Management

Q16-30
16 What is a branch in Git?
A branch is an independent line of development. It's a pointer to a specific commit, allowing you to work on features without affecting the main codebase.
17 How do you create a new branch?
git branch <branch-name> - Creates a new branch
git checkout -b <branch-name> - Creates and switches to the new branch
18 How do you switch between branches?
git checkout <branch-name> or git switch <branch-name>
19 How do you list all branches?
git branch - List local branches
git branch -a - List all (local + remote) branches
git branch -r - List remote branches only
20 How do you delete a branch?
git branch -d <branch-name> - Delete merged branch (safe)
git branch -D <branch-name> - Force delete (even if unmerged)
21 How do you rename a branch?
git branch -m <old-name> <new-name>
Or if you're on the branch: git branch -m <new-name>
22 What is the default branch in Git?
Traditionally master, but modern repositories use main as the default branch name.
23 What is a detached HEAD state?
When HEAD points directly to a commit instead of a branch. This happens when you checkout a specific commit. Any new commits won't belong to any branch.
24 How do you see which branch you're currently on?
git branch (current branch is marked with *) or git status
25 What is a feature branch?
A branch created for developing a specific feature. Once complete, it's merged back into the main branch and deleted.
26 What is Git Flow?
A branching model that uses multiple branch types: main (production), develop (integration), feature, release, and hotfix branches.
27 How do you check the differences between branches?
git diff branch1..branch2 or git diff branch1 branch2
28 What is an orphan branch?
A branch with no commit history, created using git checkout --orphan <branch-name>. Often used for GitHub Pages.
29 How do you compare your current branch with another?
git diff <other-branch>
30 What is branch pruning?
Removing references to remote branches that have been deleted. Use git fetch --prune or git remote prune origin.

Merge & Rebase

Q31-45
31 What is git merge?
Combines changes from one branch into another, creating a merge commit. Preserves complete history.
32 How do you merge a branch?
1. Switch to target branch: git checkout main
2. Merge: git merge feature-branch
33 What is a fast-forward merge?
When the target branch hasn't diverged, Git simply moves the pointer forward. No merge commit is created. Use --no-ff to force a merge commit.
34 What is git rebase?
Moves or combines commits to a new base commit, creating a linear history. Rewrites commit history.
35 What's the difference between merge and rebase?
Merge: Preserves history, creates merge commits
Rebase: Rewrites history, creates linear timeline
Rule: Never rebase public/shared branches
36 How do you perform a rebase?
1. Switch to feature branch: git checkout feature
2. Rebase onto main: git rebase main
37 What is interactive rebase?
git rebase -i HEAD~n - Allows you to edit, squash, reorder, or drop commits.
38 How do you abort a merge?
git merge --abort - Returns to state before merge started.
39 How do you abort a rebase?
git rebase --abort - Cancels the rebase operation.
40 What is cherry-pick?
git cherry-pick <commit-hash> - Applies a specific commit from one branch to another.
41 What is squash in Git?
Combining multiple commits into a single commit. Use git merge --squash or during interactive rebase.
42 When should you use merge vs rebase?
Use Merge: For public/shared branches, preserving history
Use Rebase: For local/feature branches, clean linear history
43 What is a three-way merge?
Git finds a common ancestor and merges changes from both branches, creating a merge commit.
44 How do you continue a rebase after resolving conflicts?
1. Resolve conflicts in files
2. git add <resolved-files>
3. git rebase --continue
45 What is git revert vs git reset?
revert: Creates new commit that undoes changes (safe for shared branches)
reset: Moves HEAD backward, rewriting history (use only on local branches)

Remote Repositories

Q46-55
46 What is a remote repository?
A version of your repository hosted on a server (like GitHub, GitLab). It enables collaboration.
47 How do you add a remote repository?
git remote add origin <url>
48 How do you view remote repositories?
git remote - List remotes
git remote -v - List with URLs
49 What is git fetch?
Downloads commits, files, and refs from a remote repository but doesn't merge them. Use git fetch.
50 What is git pull?
git pull = git fetch + git merge. Downloads and merges changes from remote.
51 What is git push?
Uploads local commits to a remote repository. git push origin main
52 What is origin in Git?
The default name Git gives to the remote repository you cloned from.
53 How do you push a new branch to remote?
git push -u origin <branch-name>
The -u flag sets upstream tracking.
54 How do you delete a remote branch?
git push origin --delete <branch-name>
or git push origin :<branch-name>
55 What is the difference between fetch and pull?
fetch: Downloads changes but doesn't merge
pull: Downloads and automatically merges changes

Conflict Resolution

Q56-65
56 What is a merge conflict?
Occurs when Git can't automatically resolve differences between two commits (usually when same lines are modified).
57 How do you resolve a merge conflict?
1. Open conflicted files
2. Look for conflict markers: <<<<<<<, =======, >>>>>>>
3. Edit to resolve
4. git add <file>
5. git commit
58 What do the conflict markers mean?
<<<<<<< HEAD: Your current branch changes
=======: Separator
>>>>>>> branch-name: Incoming branch changes
59 How do you see which files have conflicts?
git status - Shows "both modified" for conflicted files.
60 How do you abort a merge with conflicts?
git merge --abort
61 What tools can help resolve conflicts?
git mergetool - Opens configured merge tool
IDEs like VS Code, IntelliJ have built-in conflict resolvers
62 How do you accept all changes from one branch?
git checkout --ours <file> - Keep your changes
git checkout --theirs <file> - Accept incoming changes
63 What causes merge conflicts?
- Same lines modified in both branches
- File deleted in one branch, modified in another
- Binary file changes
64 How can you prevent merge conflicts?
- Pull frequently from main branch
- Keep branches short-lived
- Communicate with team
- Use small, focused commits
65 What is git rerere?
"Reuse Recorded Resolution" - Git remembers how you resolved conflicts and can automatically resolve similar conflicts in the future.

Advanced Topics

Q66-70
66 What is git stash?
Temporarily saves uncommitted changes. Useful when you need to switch branches.
git stash - Save changes
git stash pop - Restore and remove
git stash list - View stashes
67 What is git reflog?
Records all updates to HEAD. Useful for recovering lost commits. git reflog
68 What is git bisect?
Binary search tool to find which commit introduced a bug. git bisect start, mark commits as good/bad.
69 What is git tag?
Marks specific points in history (usually releases).
git tag v1.0.0 - Lightweight tag
git tag -a v1.0.0 -m "message" - Annotated tag
70 What are Git hooks?
Scripts that run automatically at certain Git events (pre-commit, post-commit, pre-push). Located in .git/hooks/.
