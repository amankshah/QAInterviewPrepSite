Testing Levels & Strategy

Q1-15
1 What are the different levels of testing?
1. Unit Testing: Individual components
2. Integration Testing: Module interactions
3. System Testing: Complete system
4. Acceptance Testing: User validation
2 What is the Test Pyramid and why is it important?
Bottom (Most): Unit Tests - Fast, cheap, numerous
Middle: Integration/API Tests - Moderate
Top (Least): UI/E2E Tests - Slow, expensive, few
Importance: Faster feedback, lower costs, better coverage
3 How do you create a Test Strategy?
1. Understand requirements and risks
2. Define scope and objectives
3. Identify test types needed
4. Select tools and frameworks
5. Plan resources and timeline
6. Define entry/exit criteria
7. Establish metrics and reporting
4 What is Risk-Based Testing?
Prioritizing testing based on risk assessment. High-risk areas (critical features, complex logic, frequently used) get more testing attention and resources.
5 What is Shift-Left Testing?
Moving testing activities earlier in the SDLC. Testing starts during requirements and design phases, not just after development. Catches defects when they're cheaper to fix.
6 What is Shift-Right Testing?
Testing in production environments. Includes monitoring, A/B testing, canary releases, and gathering real user feedback. Complements traditional testing.
7 How do you decide what to automate vs manual test?
Automate: Regression, smoke tests, data-driven, repetitive, stable features
Manual: Exploratory, usability, one-time tests, new features, complex scenarios
8 What is Test Coverage and how do you measure it?
Types:
- Requirements coverage: Tests per requirement
- Code coverage: Lines/branches executed
- Risk coverage: Critical areas tested
Tools: JaCoCo, Istanbul, SonarQube
9 What metrics do you track for QA?
- Test execution rate
- Pass/Fail ratio
- Defect density
- Defect leakage
- Test coverage
- Automation percentage
- Mean time to detect/fix
10 What is Continuous Testing?
Running automated tests continuously as part of CI/CD pipeline. Every code commit triggers tests, providing immediate feedback on quality.
11 How do you handle testing in Agile/Scrum?
- Test planning in sprint planning
- Write tests alongside development
- Daily testing activities
- Automate within sprint
- Demo tested features
- Retrospective on quality
12 What is Definition of Done for QA?
- All test cases executed
- No critical/high bugs open
- Code reviewed
- Automation tests added
- Documentation updated
- Performance acceptable
13 How do you estimate testing effort?
- Analyze requirements complexity
- Historical data from similar projects
- Test case count estimation
- Factor in automation time
- Include regression effort
- Add buffer for unknowns
14 What is Exploratory Testing?
Simultaneous learning, test design, and execution. Tester explores application without predefined scripts, using domain knowledge and creativity to find bugs.
15 How do you handle test documentation?
- Test plan (strategy, scope)
- Test cases (steps, data, expected)
- Traceability matrix
- Bug reports
- Test summary reports
- Keep it lean but sufficient

End-to-End Testing

Q16-30
16 What is End-to-End Testing?
Testing complete user workflows from start to finish, verifying all integrated components work together correctly. Simulates real user scenarios.
17 What should E2E tests cover?
- Critical user journeys
- Business-critical flows
- Integration points
- Data flow across systems
- Happy paths and key error scenarios
18 What are challenges in E2E testing?
- Slow execution time
- Flaky tests
- Environment complexity
- Test data management
- Debugging failures
- Maintenance overhead
19 How do you make E2E tests reliable?
- Proper wait strategies
- Stable locators
- Test isolation
- Retry mechanisms
- Clean test data
- Consistent environments
20 How do you handle test data in E2E tests?
- Create fresh data before tests
- Use API to seed data
- Database snapshots
- Cleanup after tests
- Avoid shared test data
- Use data factories
21 How do you test login/authentication E2E?
- Valid credentials login
- Invalid credentials (error messages)
- Session management
- Remember me functionality
- Password reset flow
- Multi-factor authentication
- Logout functionality
22 How do you test e-commerce checkout flow?
1. Add products to cart
2. View/edit cart
3. Apply promo codes
4. Enter shipping details
5. Select payment method
6. Complete payment
7. Order confirmation
8. Email notification
23 How do you test search functionality?
- Exact match search
- Partial match
- No results handling
- Special characters
- Filters and sorting
- Pagination
- Auto-suggestions
- Search performance
24 How do you test file upload functionality?
- Valid file types
- Invalid file types (rejection)
- File size limits
- Multiple file upload
- Upload progress
- Cancel upload
- File preview
25 How do you test form validation?
- Required fields
- Field format validation (email, phone)
- Min/max length
- Error message display
- Client-side vs server-side
- Tab order/navigation
26 How do you test pagination?
- First/last page navigation
- Next/previous buttons
- Page number links
- Items per page
- Total count accuracy
- Empty page handling
- URL parameters
27 How do you test email functionality?
- Email delivery
- Subject and content
- Links in email work
- Attachments
- Spam filtering
- Use tools like Mailinator, Mailtrap
28 How do you test notifications/alerts?
- Trigger conditions
- Display timing
- Content accuracy
- Dismiss functionality
- Multiple notifications
- Push notifications
- Notification preferences
29 How do you test user roles and permissions?
- Different role access levels
- Feature visibility per role
- Action restrictions
- Role switching
- Unauthorized access attempts
- Admin vs regular user
30 How do you handle third-party integrations in E2E?
- Mock external services when possible
- Use sandbox/test environments
- Test happy path with real integration
- Test error handling with mocks
- Contract testing

API Testing

Q31-45
31 What is API Testing?
Testing application programming interfaces directly. Validates functionality, reliability, performance, and security of APIs without UI.
32 What are HTTP methods and when to use them?
GET: Retrieve data
POST: Create new resource
PUT: Update entire resource
PATCH: Partial update
DELETE: Remove resource
33 What HTTP status codes should you verify?
2xx Success: 200 OK, 201 Created, 204 No Content
4xx Client Error: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found
5xx Server Error: 500 Internal Server Error
34 What do you validate in API response?
- Status code
- Response body (JSON structure)
- Response headers
- Response time
- Data types
- Required fields
- Error messages
35 What is REST API?
Representational State Transfer - architectural style using HTTP methods, stateless communication, resource-based URLs, and JSON/XML data format.
36 What is GraphQL and how is it different from REST?
GraphQL: Query language where client specifies exact data needed. Single endpoint, no over/under fetching.
REST: Multiple endpoints, fixed data structure per endpoint.
37 How do you test API authentication?
- Valid token access
- Invalid/expired token rejection
- Missing token handling
- Token refresh flow
- OAuth flows
- API key validation
38 What are common API security tests?
- SQL injection
- Authentication bypass
- Authorization checks
- Rate limiting
- Input validation
- Sensitive data exposure
- CORS configuration
39 How do you test API with different data?
- Valid data (happy path)
- Invalid data types
- Boundary values
- Empty/null values
- Special characters
- Large payloads
- Missing required fields
40 What tools do you use for API testing?
Manual: Postman, Insomnia, cURL
Automation: RestAssured, Karate, pytest, SuperTest
Performance: JMeter, Gatling, k6
41 What is JSON Schema validation?
Validating JSON response structure against a schema definition. Ensures correct data types, required fields, and format compliance.
42 How do you handle API dependencies in testing?
- Create dependent data via API
- Use setup/teardown
- Mock dependencies
- Use test containers
- Chain API calls properly
43 What is Contract Testing?
Verifying API provider and consumer agree on contract (request/response format). Tools: Pact, Spring Cloud Contract. Catches breaking changes early.
44 How do you test API performance?
- Response time under load
- Throughput (requests/second)
- Concurrent user handling
- Resource utilization
- Error rate under stress
45 How do you document API tests?
- Clear test case names
- Request/response examples
- Expected outcomes
- Test data requirements
- Setup/prerequisites
- Postman collections

Database Testing

Q46-55
46 What is Database Testing?
Validating data integrity, schema, stored procedures, triggers, and data manipulation operations in the database.
47 What do you test in database testing?
- Data integrity
- Data accuracy
- CRUD operations
- Schema validation
- Stored procedures
- Triggers
- Data migration
48 How do you verify data after UI actions?
1. Perform action via UI/API
2. Query database directly
3. Compare expected vs actual values
4. Verify all affected tables
5. Check timestamps, foreign keys
49 What SQL queries are important for testing?
- SELECT (verify data)
- INSERT (test creation)
- UPDATE (test modifications)
- DELETE (test removal)
- JOIN (verify relationships)
- COUNT/SUM (verify aggregations)
50 How do you test data integrity?
- Primary key uniqueness
- Foreign key relationships
- NOT NULL constraints
- Default values
- Check constraints
- Referential integrity
51 How do you test stored procedures?
- Execute with valid inputs
- Execute with invalid inputs
- Verify output/return values
- Check affected data
- Test error handling
- Performance testing
52 How do you test database triggers?
- Verify trigger fires on event
- Check triggered actions executed
- Test before/after triggers
- Validate data changes
- Test with cascade operations
53 What is data migration testing?
- Source vs target data comparison
- Record count verification
- Data transformation accuracy
- No data loss
- Rollback capability
- Performance during migration
54 What tools do you use for database testing?
- SQL clients (DBeaver, DataGrip)
- DBUnit
- TestContainers
- Liquibase/Flyway
- Custom queries in test scripts
55 How do you handle test database setup?
- Separate test database
- Reset before test suite
- Seed with test data
- Use transactions for rollback
- Docker containers for isolation

Behavioral & Situational Questions

Q71-85
71 How do you handle tight deadlines with incomplete testing?
- Prioritize critical test cases
- Risk-based testing approach
- Communicate status clearly
- Document untested areas
- Propose post-release testing
- Negotiate scope if possible
72 How do you handle disagreements with developers about bugs?
- Provide clear reproduction steps
- Share evidence (screenshots, logs)
- Reference requirements
- Stay professional
- Involve team lead if needed
- Focus on user impact
73 Describe a critical bug you found and how you handled it.
Structure your answer:
1. Context/situation
2. How you discovered it
3. Impact assessment
4. Communication steps
5. Resolution and outcome
6. Lessons learned
74 How do you prioritize test cases with limited time?
- Business-critical features first
- High-risk areas
- Most frequently used features
- Customer-facing functionality
- New/changed features
- Integration points
75 How do you stay updated with QA trends?
- Tech blogs and newsletters
- Online courses
- QA communities
- Conferences/webinars
- Practice with new tools
- Side projects
76 How do you handle testing with unclear requirements?
- Ask clarifying questions
- Document assumptions
- Create test scenarios for review
- Use exploratory testing
- Test common user expectations
- Get sign-off on test approach
77 Tell me about a time you improved the testing process.
Use STAR method:
S: Situation - What was the problem?
T: Task - What needed improvement?
A: Action - What did you do?
R: Result - What was the impact?
78 How do you handle production issues?
- Stay calm
- Reproduce if possible
- Gather information (logs, errors)
- Communicate to stakeholders
- Help with root cause analysis
- Add test coverage to prevent recurrence
79 How do you ensure quality when working remotely?
- Clear communication channels
- Regular sync meetings
- Detailed documentation
- Screen sharing for demos
- Proper tool access
- Proactive status updates
80 How do you mentor junior QA team members?
- Pair testing sessions
- Code reviews for test scripts
- Share knowledge resources
- Give constructive feedback
- Assign progressively challenging tasks
- Create safe learning environment
81 What's your approach to learning a new application?
- Review documentation
- Talk to developers/PO
- Explore as end-user
- Understand architecture
- Identify integrations
- Note questions for team
82 How do you handle testing a feature you don't agree with?
- Share feedback professionally
- Document concerns with evidence
- Respect final decisions
- Test thoroughly regardless
- Focus on quality not preferences
83 What's your biggest testing failure and what did you learn?
- Be honest about the mistake
- Explain the context
- Focus on lessons learned
- Describe process improvements
- Show growth from experience
84 How do you balance thoroughness with speed?
- Smart test prioritization
- Risk-based approach
- Automation for regression
- Efficient test design
- Reusable test data
- Know when "good enough"
85 Where do you see yourself in 5 years as a QA?
Options to consider:
- Senior QA/Lead role
- QA Manager
- SDET/Automation architect
- Performance engineering
- Security testing specialist
- DevOps/Platform engineering
