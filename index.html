1. Core QA Testing Terminology & Life Cycles

Concept Key Definition / Takeaway
Software Testing Process of evaluating and verifying that software meets specified requirements and works as intended to deliver business value.
Verification "Are we building the product right?" Ensures product conforms to specifications. It is a static process (e.g., reviews, inspections).
Validation "Are we building the right product?" Ensures the product meets user/customer needs. It is a dynamic process (e.g., execution, UAT).
SDLC (Software Development Life Cycle) The entire process: Planning, Design, Implementation, Testing, Deployment, Maintenance.
STLC (Software Testing Life Cycle) The testing phase cycle: Requirements Analysis, Test Planning, Test Case Development, Environment Setup, Execution, Closure.
Black Box Testing Testing the functionality without knowing the internal code structure (Behavioral Testing).
White Box Testing Testing the internal code structure and logic (Structural Testing).
Non-Functional Requirement Specifies how the system SHOULD BE (e.g., Performance, Security, Reliability, Usability).

2. Defect Management: Priority vs. Severity

Attribute Definition Perspective Common High Levels
Severity The seriousness or technical impact of the bug on the system's functionality. Application POV Critical: App crash/data loss. High: Major function fails.
Priority The order in which the bug should be fixed, based on business impact and urgency. User/Business POV Immediate: Must be fixed in the next build. High: Should be fixed soon.

Example Scenarios

Scenario Severity Priority Reasoning
Payment button is slightly misaligned (cosmetic). Low High Low technical impact, but high business urgency (affects user checkout).
Application crashes on a hidden admin report page. Critical Low High technical impact, but low user exposure/business urgency.

3. Git & GitHub Essential Commands

Command Category Command Purpose
Setup git init Initializes a new local Git repository.
Staging git add . Stages all modified and new files for the next commit.
Commit git commit -m "message" Records the staged snapshot permanently in the history.
Branching git checkout -b <branch-name> Creates a new branch and switches to it.
Synchronization git pull Fetches and merges changes from the remote repository.
Undoing (Safe) git revert <commit-hash> Creates a new commit that reverses the changes of a prior commit. Use for public branches.
Undoing (Destructive) git reset --hard HEAD~1 Discards the last commit AND any local changes. Rewrites history. Use for local, un-pushed commits.

Action / Feature Playwright Command (JavaScript) Key Detail / Purpose
Navigation await page.goto('url'); Navigates to a URL. Waits for the page to load by default.

Handling Popups page.on('popup', popup => { /* handle popup */ }); Used to handle new browser windows/tabs.

7. Essential JavaScript Interview Concepts ðŸ“œ

Concept Simple Definition / Answer
Closure A function that retains access to its lexical scope (variables from its outer function), even after the outer function has finished execution.
var, let, const var is function-scoped; let and const are block-scoped. const cannot be re-assigned.
== vs. === == (Loose) checks value after performing type coercion. === (Strict) checks for both value AND type. Always use ===.
Hoisting The behavior where all declarations (of variables and functions) are moved to the top of their scope during compilation.
Promise An object representing the eventual completion or failure of an asynchronous operation. States: pending, fulfilled, or rejected.
async/await Syntax built on Promises that makes asynchronous code look synchronous. async defines a function; await pauses execution until the Promise resolves.
Closure Use Case Common uses: Data privacy (private variables), Function factories, Event handlers, and Memoization. Example: function counter() { let count = 0; return () => ++count; }
map() Creates a new array by applying a function to each element. Does NOT mutate the original. Example: [1,2,3].map(x => x * 2) returns [2,4,6].
filter() Creates a new array with elements that pass a test (return true). Example: [1,2,3,4].filter(x => x > 2) returns [3,4].
reduce() Reduces an array to a single value by applying a function. Takes an accumulator and current value. Example: [1,2,3].reduce((acc, x) => acc + x, 0) returns 6.
forEach() vs map() forEach executes a function for each element but returns undefined. map returns a new array. Use map when you need the transformed result.
find() / findIndex() find() returns the first element that matches. findIndex() returns the index of the first match, or -1 if not found.
some() / every() some() returns true if at least one element passes the test. every() returns true only if all elements pass.
Arrow Functions Shorter syntax: const add = (a, b) => a + b;. Key difference: Arrow functions do NOT have their own this; they inherit from the enclosing scope.
Spread Operator (...) Expands iterables into individual elements. Use cases: Copy arrays [...arr], Merge objects {...obj1, ...obj2}, Function args func(...args).
Destructuring Extract values from arrays/objects. Array: const [a, b] = [1, 2]; Object: const {name, age} = person;. Great for cleaner code.
Higher-Order Function A function that takes a function as argument OR returns a function. Examples: map, filter, reduce, custom decorators.
Callback Function A function passed as an argument to another function, executed after some operation. Example: setTimeout(() => console.log('Hi'), 1000);
Method Chaining Calling multiple methods in sequence. Example: arr.filter(x => x > 0).map(x => x * 2).reduce((a, b) => a + b, 0)

8. Advanced/Tricky QA Interview Questions ðŸ§ 

Question Recommended Answer Framework
How do you handle a bug a Developer claims is "Not a Bug"? 1. Re-verify and replicate. 2. Document clearly (video/logs). 3. Cite Requirement or specific business logic being violated. 4. Escalate to PO/QA Lead for decision.
Difference between Git Merge and Git Rebase? Merge: Preserves history, creates a merge commit. Rebase: Rewrites history for a clean, linear log. Rule: Rebase only on local branches; Merge on public/shared branches.
Explain Playwright's Actionability Checks. Playwright automatically checks if an element is Visible, Enabled, and Stable before executing an action. This prevents common race conditions and flaky tests.
How do you test Search Functionality? Test Functional (exact/partial match, negative testing), Performance (search speed), and Usability (autocomplete, relevance/ranking).
